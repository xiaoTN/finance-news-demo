# 金融新闻 AI 分析师 - 项目说明

## 项目目标
- 最终做成可桌面化的软件产品，但当前阶段先完成 **Web Demo**。
- 全自动实时抓取金融新闻与关键人物动态。
- 用 AI 大模型作为分析大脑，输出：
  - 利好谁
  - 利空谁
  - 为什么（从宏观政策、机构态度、公司基本面给出因果解释）

## 需要解决的核心痛点
1. 最新信息获取不及时
- 重点是美股一手信息，来源多且分散。
- 降低新手用户获取信息的门槛与延迟。

2. 信息到股票影响的分析能力不足
- 需要把新闻影响拆解为：宏观政策、华尔街态度、公司基本面。
- 把噪音信息转成可执行信号，并给出清晰原因链条。

## 数据源
- RSS: Reuters、CNBC、MarketWatch、Benzinga、Seeking Alpha、Federal Reserve
- JSON: NVIDIA Newsroom
- 模拟数据: 雪球、金十数据
- 关键人物：特朗普、马斯克、黄仁勋、鲍威尔

## 当前阶段 Demo 范围
- 以 Web 应用形式实现：
  - 自动抓取链路（支持 RSS/JSON/模拟数据）
  - 事件标准化与去重
  - 人物/实体识别与股票代码映射
  - AI 分析输出（`impact`、`why`、`horizon`）
  - 实时事件看板（支持按发布时间/抓取时间排序）

## 每条事件的输出字段规范
- `title`：标题
- `source_name`：来源名称
- `published_at`：原始发布时间
- `captured_at`：抓取时间
- `persons`：涉及人物
- `tickers`：影响标的代码
- `summary`：摘要
- `impact`：`利好` | `利空` | `mixed`
- `why`：因果解释
- `horizon`：`intraday` | `swing` | `long_term`
- `url`：原文链接

## 实现原则
- 先保证 Demo 可运行，尽量减少复杂依赖。
- 使用本地 SQLite，便于快速迭代。
- 架构预留桌面化能力（后续可对接 Tauri/Electron）。

## 协作与提交流程
- 每次代码或配置改动后，必须立即执行一次 Git commit。
- 每次完成功能后都必须验证功能是否正常工作。
- 默认不执行 push，除非用户明确要求。
